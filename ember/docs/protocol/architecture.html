Ember Protocol Architecture

Android app ↔ ember-server ↔ grpc_server — QUIC, gRPC, and push channels

📱 Android App
QUIC client
Connect: host:4433
Send: prompt (raw UTF-8)
Receive: newline-JSON stream
Control: __fetch_push__, __get_style__, __check_in__
⟷ QUIC/UDP 4433
🖥️ ember-server
0.0.0.0:4433 QUIC
0.0.0.0:4434 TCP push
Control pipe: __word__ in-process
User prompt: → gRPC inference
proactive_queue + push_notify
→ gRPC 50051
⚙️ grpc_server
127.0.0.1:50051
complete_stream / complete
LLM inference (llama.cpp)
📤 Push Sources
TCP 4434: External processes, one line = one message
push-queue.txt: File poll (1s)
Proactive task: 30 min check-in → LLM
Readiness: "Ready for inference!"
AI output: <ember_push>, __command__
→ proactive_queue
📥 App Retrieval
App sends __fetch_push__
Server long-polls (60s) until queue has msg
Returns payload immediately when pushed

Request Flow (User Prompt)

  • User types prompt → App sends over QUIC stream
  • Server: not control message → format_prompt → gRPC complete_stream
  • grpc_server streams tokens → ember parses <ember_*>, __command__
  • Server sends stream_start, stream_token, stream_rich, stream_end
  • Client parses JSON lines, invokes TokenCallback (onToken, onRichContent, etc.)

Push Flow

  • External: TCP connect to 4434, write "Hello" or "marquee" or JSON
  • Server: push to proactive_queue, push_notify.notify_one()
  • App: __fetch_push__ long-poll wakes, pops message, returns to app
  • App: applyPushPayload() → update chat, rich, layout, or "app clear"
