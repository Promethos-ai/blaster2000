Ember Stream Frames

Newline-delimited JSON frames (Server → App). Client parses and invokes TokenCallback.

stream_start {"type":"stream_start","interaction_id":N} Ignore
stream_token {"type":"stream_token","token":"...","interaction_id":N} onToken() → chat
stream_rich {"type":"stream_rich","content":"<div>...</div>","interaction_id":N} onRichContent() → rich area
stream_style {"type":"stream_style","css":"...","interaction_id":N} onStyle() → rich CSS
stream_layout {"type":"stream_layout","json":"{...}","interaction_id":N} onLayout() → height, timeout
stream_audio {"type":"stream_audio","text":"...","interaction_id":N} onAudio() → TTS
stream_control_payload {"type":"stream_control_payload","payload":"app clear","interaction_id":N} onControlPayload() → applyPushPayload
stream_end {"type":"stream_end","interaction_id":N} Finish, return result
stream_error {"type":"stream_error","error":"...","interaction_id":N} Show error, stop

Frame Type Reference

Type Key Fields Client Callback
stream_token token onToken — append to last AI message, batch 80ms
stream_rich content (HTML) onRichContent — RichContentWebView.updateContent
stream_style css onStyle — RichContentWebView.applyStyle
stream_layout json (rich_height, inference_timeout_sec) onLayout — applyLayout
stream_audio text onAudio — speakText (TTS)
stream_control_payload payload onControlPayload — applyPushPayload (e.g. "app clear")
Legacy mode: If the server sends raw text (no JSON), the client switches to legacy mode: accumulates the entire response and treats it as a single token stream.
