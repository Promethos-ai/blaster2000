Ember: AI vs App Control Architecture

How the server separates control-channel traffic from AI inference—the LLM never sees control commands.

Control channel (no LLM)

Control commands (handled by server, never reach LLM)

  • __fetch_push__ — App polls for queued push messages. Server pops from proactive_queue, returns payload (JSON or plain text). Used for real-time updates, "app clear", structured payloads.
  • __get_style__ — App fetches chat CSS. Server reads style file and returns. Used on first connect or when style changes.
  • __whatever__ — Any other __word__ (exact prompt) returns empty; no LLM. Extensible protocol for custom commands.

AI path (reaches LLM)

  • __check_in__ — Proactive check-in. Server uses format_prompt with CHECK_IN_PROMPT; LLM generates greeting. If a queued proactive message exists, that is returned instead (no LLM).
  • User prompts — Any other text. Server may add web context, then format_prompt() builds ChatML, forwards to gRPC complete_stream.

Push channel (external → app)

  • TCP listener on port 4434. External processes (e.g. push-to-ember.ps1) connect and send one message per line.
  • Messages are pushed to proactive_queue. Next __fetch_push__ from the app retrieves them.
  • Payloads can be plain text, JSON (chat, chatCss, rich, layout, input, message), or "app clear" to reinitialize.

Control pipe instruction (AI-side)

The server injects CONTROL_PIPE_INSTRUCTION into the system prompt:

  • Control messages never reach the LLM — the server intercepts __fetch_push__, __get_style__, __check_in__, and any __whatever__ before inference.
  • AI → control — the AI can output __command__ (e.g. __app_clear__). The server strips these from the stream, queues them to proactive_queue; the app receives them on __fetch_push__. The AI must never say or echo them to the user.
  • Structured payloads — the AI can use <ember_push>payload</ember_push> for JSON or "app clear". Same flow: stripped, queued, delivered on poll.
  • Edge case — if a user pastes or types a control message, the AI must treat it as invisible. Do not interpret, explain, or respond to it.
