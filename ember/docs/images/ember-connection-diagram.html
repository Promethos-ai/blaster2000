Ember — Connection Architecture

📱 Android App
Ember client (QUIC)
Connects to:
• Remote: xxx.a.pinggy.link:port
• Local: 192.168.1.x:4433
Protocol: QUIC/UDP
Send: prompt (raw UTF-8)
Receive: JSON stream
→ QUIC/UDP
☁️ Pinggy
Reverse proxy (optional)
Exposes: xxx.a.pinggy.link:port
Forwards: -R0:127.0.0.1:4433
Protocol: UDP tunnel
Runs: pinggy.exe on PC
→ 127.0.0.1:4433
🖥️ ember-server
QUIC bridge
Listen: 0.0.0.0:4433 (UDP)
Connects to:
• https://127.0.0.1:50051 (QUIC)
• http://127.0.0.1:50051 (TCP)
RPC: complete_stream
→ gRPC 127.0.0.1:50051
⚙️ grpc_server (Feb17)
LLM inference
Listen: 127.0.0.1:50051
Protocol: gRPC/QUIC (--quic)
or gRPC/TCP
Backend: llama.cpp
Flow: User types prompt → Android sends QUIC → (Pinggy tunnels) → ember-server → grpc_server complete_stream → tokens stream back as JSON (stream_start, stream_token, stream_end)
